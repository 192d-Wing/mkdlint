## [Unreleased]

## [0.11.6] - 2026-02-17

### Added

- **JSON Schema published** at `schema/mkdlint-schema.json` in the repository;
  the `$schema` URL generated by `mkdlint init` and `mkdlint --generate-schema`
  now resolves to a real file for editor validation
- **`LintError::fix_only` field**: internal fix-only helper errors (e.g. MD003
  setext underline deletion, MD046 block line deletion) now carry a dedicated
  `fix_only: bool` flag instead of empty `rule_names`/`rule_description`
  sentinels; all formatters, LSP diagnostics, and counts skip `fix_only` errors
- **`--fix-dry-run` integration tests**: three new e2e tests verify that
  `--fix-dry-run` exits 1 when fixable issues exist, exits 0 when clean, and
  never modifies files on disk
- **LSP hover alias tests**: two new LSP integration tests verify that hovering
  over a rule alias (`no-missing-space-atx`) or canonical name (`MD009`) in a
  document returns rule documentation even when no diagnostic exists on the line

### Fixed

- **SARIF panic** on files triggering MD003/MD046 setext‚ÜíATX conversion: the
  SARIF formatter now skips `fix_only` errors rather than those with empty
  `rule_names` (the underlying wart is now eliminated)
- **SARIF upload rejected** by GitHub Code Scanning: empty `helpUri` is now
  omitted; `deletedRegion.endColumn` no longer uses `usize::MAX`; ghost errors
  no longer produce results with empty `message.text`
- **SARIF fix `artifactChanges`**: fix objects now include the required
  `artifactChanges` property with proper `deletedRegion` and `insertedContent`
- **Windows CI**: added `.gitattributes` (`* text=auto eol=lf`) to prevent CRLF
  conversion from breaking insta snapshot tests; LSP path tests guarded with
  `#[cfg(unix)]`
- **action.sh version capture**: `info/success/warn` logging functions now write
  to stderr so command substitution captures are not polluted with ANSI text
- **Cargo.lock committed** for reproducible binary builds; its absence caused
  Docker build failures in the Release workflow
- **deny.toml**: added `CC0-1.0` to allowed licenses (used by `notify` crate)
- **Debouncer test** flakiness on slow macOS CI: sleep increased 100ms ‚Üí 500ms

## [0.11.0] - 2026-02-17

### Added

- **Kramdown preset** (`--preset kramdown`) for RFC and technical document authors:
  - Activatable via `--preset kramdown` CLI flag or `"preset": "kramdown"` in config
  - Disables MD033 (inline HTML ‚Äî conflicts with IAL `{: #id .class}` syntax) and
    MD041 (first heading ‚Äî RFC preambles often start with metadata)
  - Enables 10 new Kramdown-specific lint rules (KMD001‚ÄìKMD010, off by default)
- **GitHub preset** (`--preset github`) for GitHub-hosted documentation:
  - Sets MD003 heading style to `consistent`
  - Disables MD013 (line length) and MD034 (bare URLs) ‚Äî both common in GFM docs
- **KMD001** (`definition-list-term-has-definition`): Definition list terms must be
  followed by a `: definition` line
- **KMD002** (`footnote-refs-defined`): Footnote references `[^label]` must have
  matching `[^label]:` definitions
- **KMD003** (`footnote-defs-used`): Footnote definitions must be referenced in
  the document body
- **KMD004** (`abbreviation-defs-used`): Abbreviation definitions `*[ABBR]: ...`
  must appear as text in the document
- **KMD005** (`no-duplicate-heading-ids`): Heading IDs (explicit `{#id}` or
  auto-generated slugs) must be unique; detects both ATX and setext headings;
  **auto-fixable** ‚Äî appends `{#slug-N}` suffix to disambiguate
- **KMD006** (`valid-ial-syntax`): Block-level IAL lines `{: ...}` must be
  well-formed; **auto-fixable** ‚Äî deletes the malformed IAL line
- **KMD007** (`math-block-delimiters`): Block math `$$` fences must have a
  matching closing `$$`; **auto-fixable** ‚Äî inserts closing `$$` at EOF
- **KMD008** (`block-extension-syntax`): Block extensions `{::name}...{:/name}`
  must be properly opened and closed; self-closing `{::name .../}` is valid;
  **auto-fixable** ‚Äî inserts closing `{:/name}` at EOF
- **KMD009** (`ald-defs-used`): Attribute List Definitions `{:ref-name: attrs}`
  must be referenced somewhere in the document; **auto-fixable** ‚Äî deletes the
  unused ALD definition line
- **KMD010** (`inline-ial-syntax`): IAL syntax used inline on spans
  (e.g. `*text*{: .class}`) must be well-formed; **auto-fixable** ‚Äî removes the
  malformed IAL span
- **`--preset` CLI flag**: apply a named rule preset from the command line
- **`--list-presets`**: show all available presets and their rule changes
- **`--list-rules` improvements**: preset annotations (‚óè enabled / ‚óã disabled),
  KMD rules dimmed when off-by-default, MD/KMD groups separated, fixable count in footer
- **LSP**: `mkdlint.preset` VS Code workspace setting ‚Äî set the active preset
  without a config file; hot-reloads when the setting changes
  (`workspace/didChangeConfiguration`)
- **Auto-fix coverage**: 54/63 rules now fixable (85.7%), up from 48/63

### Fixed

- **KMD006**: Block extension lines (`{::name}`, `{:/name}`) and ALD definition
  lines (`{:id: ...}`) were incorrectly flagged as malformed IAL ‚Äî now excluded
- **MD023**: Panic on multi-byte UTF-8 heading context truncation ‚Äî replaced
  byte-index slice with char-boundary-safe `char_indices().nth(20)` slicing
